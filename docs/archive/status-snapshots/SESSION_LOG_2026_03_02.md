## 技術的決定事項

### 決定1: OpenSpec アーカイブ戦略

**問題:** `ui-stability-and-cleanup` のアーカイブ時に、新規specが存在しないためエラー

**決定:** `--skip-specs` フラグで一旦アーカイブし、spec作成を後続タスクで実施

**理由:**
- アーカイブ自体は完了状態を記録する重要な作業
