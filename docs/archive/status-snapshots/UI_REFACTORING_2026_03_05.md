## 改修の背景

### 課題

プロジェクト評価の結果、UIが複雑化していることが判明：

1. **ツールバーのボタン過多：** 16個のUI要素（ボタン＋セレクト）
2. **フローティングパネルの乱立：** 6個の独立したパネル
3. **FABボタンの冗長性：** 2個のFABボタン（機能重複）
4. **サイドバーの複雑性：** 4タブ、26個のガジェット、配置が無秩序

### 目標

Novlr、Obsidian、Scrivenerを参考に、シンプル・モダンなUIに整理：

- ツールバーを基本機能のみに絞る（11個）
- フローティングパネルを1つに統合
- FABボタンを削除
- サイドバーを1タブ化（次回実施）

### Phase 1: FABボタン削除

**削除対象：**
- `show-toolbar`（ツールバー再表示用FAB）
- `fab-tools`（ツールパネル用FAB）

**理由：**
- ツールバーにボタンを統合できる
- ショートカットキーで代替可能
- 画面スペースの無駄

**変更ファイル：**
- `index.html` - FABボタンHTML削除
- `js/element-manager.js` - FABボタン参照削除

### Phase 3: フローティングパネル統合

**統合前（6個の独立パネル）：**

1. `floating-font-panel`（クイックツール）
2. `search-panel`（検索/置換）
3. `font-decoration-panel`（フォント装飾）
4. `text-animation-panel`（テキストアニメーション）
5. `global-search-panel`（全文検索）
6. `split-view-mode-panel`（分割ビューモード）

**統合後（1個のメインハブパネル）：**

`main-hub-panel` - タブ形式で6機能を統合

**タブ構成：**
1. クイックツール - エディタフォントサイズ、HUD制御
2. 検索 - テキスト検索/置換、正規表現対応
3. 装飾 - 太字、斜体、下線など15種類の装飾
4. アニメーション - 7種類のアニメーション効果、設定調整
5. 全文検索 - 全ドキュメント横断検索、履歴管理
6. 分割ビュー - 編集/プレビュー、章間比較、差分表示

**新機能：**
- タブクリックで機能切り替え
- ツールバーボタンから対応するタブを直接表示
- ドラッグ&ドロップで位置調整
- レスポンシブ対応（モバイルでは2列レイアウト）
- アクセシビリティ対応（ARIA属性）

**変更ファイル：**
- `index.html` - メインハブパネル追加、既存パネルコメントアウト
- `css/style.css` - タブUIスタイル追加
- `js/main-hub-panel.js` - タブ切り替えロジック（新規）
- `js/modules/editor/EditorUI.js` - ボタンイベント更新
- `js/modules/editor/EditorSearch.js` - 検索パネル統合

## ユーザー影響

### プラスの影響

1. **シンプルなUI：** ボタンが減り、視覚的にすっきり
2. **統一感：** パネルが1つに統合され、操作が直感的
3. **画面スペース：** FABボタン削除で編集エリアが広がる
4. **アクセシビリティ：** ARIA属性追加で支援技術対応向上

### マイナスの影響

1. **学習コスト：** タブUIに慣れる必要がある
2. **ワンクリック削除：** 一部機能がタブ内に移動（1クリック増加）

### 互換性

- 既存のショートカットキーはすべて動作
- LocalStorageのデータは影響なし
- 既存の個別パネルはコメントアウトで保持（後方互換性）

## 技術的詳細

### main-hub-panel.js

**主要機能：**
- タブ切り替えロジック
- パネル表示制御
- 特定タブへの直接切り替えAPI

**グローバルAPI：**
```javascript
window.MainHubPanel = {
  show: function(tabName) { ... },
  hide: function() { ... },
  switchTab: function(tabName) { ... }
};
```

**使用例：**
```javascript
// メインハブパネルを開いて装飾タブを表示
MainHubPanel.show('decoration');

// 検索タブに切り替え
MainHubPanel.switchTab('search');

// パネルを閉じる
MainHubPanel.hide();
```

### CSSスタイル

**新規追加クラス：**
- `.main-hub-panel` - メインハブパネル本体
- `.panel-tabs` - タブコンテナ
- `.panel-tab` - タブボタン
- `.panel-tab.active` - アクティブタブ
- `.tab-content` - タブコンテンツ
- `.tab-content.active` - 表示中のコンテンツ

**レスポンシブ対応：**
```css
@media (max-width: 768px) {
  .panel-tabs {
    grid-template-columns: repeat(2, 1fr); /* 2列レイアウト */
  }
}
```

## Phase 4: サイドバーアコーディオン化（実装完了）

**実施日**: 2026年3月5日
**コミット**: 5741b69, 2c99d78, 23520dd

### 実装内容

#### HTML構造の変更

- `.sidebar-accordion`ラッパー導入
- 5つの`.accordion-category`セクション（structure/edit/theme/assist/advanced）
- `.accordion-header`（クリック可能なヘッダー）
- `.accordion-content`（折りたたみ可能なコンテンツ）
- 各カテゴリに対応する`gadgets-panel`

#### JavaScript実装

- `SidebarManager.bootstrapAccordion()`メソッド追加
- `_toggleAccordion()`, `_setAccordionState()`で開閉制御
- LocalStorageに展開状態を保存（`accordionState`）
- ARIA属性による完全なアクセシビリティ対応

#### CSS実装

- アコーディオン専用スタイル（約100行）
- スムーズな開閉アニメーション（max-height transition）
- レスポンシブ対応（768px以下で調整）
- ダークモード対応

#### ガジェット再配置

- 30個のガジェットを5カテゴリに分類
- 16ファイルのgroups設定を更新
- 旧グループ（wiki/typography/settings）から新グループへ移行

### Before/After比較

| 項目           | Before   | After        | 改善              |
| -------------- | -------- | ------------ | ----------------- |
| タブ数         | 4タブ    | 5カテゴリ    | アコーディオン化   |
| 同時表示       | 1タブのみ | 複数カテゴリ | 柔軟性向上        |
| ガジェット配置 | 曖昧     | 論理的分類   | 使いやすさ向上    |

### ユーザー影響

**プラス**:

- 複数カテゴリの同時展開が可能
- ガジェットの配置が論理的で見つけやすい
- 展開状態が自動保存される

**マイナス**:
