# SP-063 Markdownリファレンス仕様 v1.0

## 目的

執筆中にMarkdown記法・ショートカット・アプリ固有拡張をすぐ参照できるリファレンスガジェットを提供し、学習コストを下げる。

## 背景

- 現在のヘルプ（`gadgets-help.js`）はモーダル型で操作手順中心。Markdown記法リファレンスは Ctrl+B/I/K の3件のみ。
- Story Wiki（SP-050）は物語用語専用であり、技術リファレンスの提供場所がない。
- WYSIWYG モード（SP-055）導入でMarkdownを直接書かないユーザーも増えるが、ツールバーボタンとMarkdown記法の対応表があると理解が深まる。

## 設計方針

- **ガジェット形式**: `assist` グループに `MarkdownReference` として登録
- **静的コンテンツ**: ユーザー編集不要。アプリ組み込みの定義済みリファレンス
- **カテゴリ分類 + 検索**: カテゴリ別折りたたみ + テキスト検索で絞り込み
- **コンパクト表示**: サイドバー内に収まるよう各項目は1-2行で簡潔に

## コンテンツ構成

### カテゴリと項目

1. **基本装飾**
   - `**太字**` → **太字** (`Ctrl/⌘+B`)
   - `*斜体*` → *斜体* (`Ctrl/⌘+I`)
   - `~~取り消し線~~` → ~~取り消し線~~
   - `` `インラインコード` `` → `インラインコード`

2. **見出し**
   - `# 見出し1` 〜 `###### 見出し6`
   - 見出しレベルと構造の推奨（章=H1, シーン=H2, 小見出し=H3）

3. **ブロック要素**
   - 順序なしリスト: `- 項目` / `* 項目`
   - 順序付きリスト: `1. 項目`
   - 引用: `> 引用テキスト`
   - コードブロック: `` ``` ``
   - 水平線: `---`

4. **リンク・画像**
   - リンク: `[テキスト](URL)` (`Ctrl/⌘+K`)
   - 画像: `![代替テキスト](URL)`

5. **テーブル**
   - パイプ構文: `| 列1 | 列2 |`

6. **アプリ固有拡張**
   - Wikiリンク: `[[用語名]]`
   - ルビ: `{漢字|かんじ}` （SP-059）
   - 装飾プリセット: `:::preset-name` （SP-060）

7. **キーボードショートカット**
   - 装飾: Ctrl+B / I / K
   - 元に戻す/やり直し: Ctrl+Z / Ctrl+Shift+Z
   - 検索: Ctrl+F
   - 保存: Ctrl+S
   - WYSIWYG切替: Ctrl+Shift+W（該当する場合）

8. **WYSIWYGとの対応**
   - ツールバーボタン → Markdown記法の対応表
   - ブロック操作（見出し/リスト/引用）の操作方法

## UI設計

### ガジェット構造

```
┌─ Markdownリファレンス ──────────────┐
│ [🔍 検索...]                        │
│                                      │
│ ▼ 基本装飾                           │
│   **太字**     Ctrl+B                │
│   *斜体*       Ctrl+I                │
│   ~~取消線~~                         │
│                                      │
│ ▶ 見出し                             │
│ ▶ ブロック要素                       │
│ ▶ リンク・画像                       │
│ ▶ アプリ固有拡張                     │
│ ▶ ショートカット                     │
└──────────────────────────────────────┘
```

### 検索動作

- 入力文字列でカテゴリ内の全項目をフィルタリング
- マッチしたカテゴリは自動展開、マッチしないカテゴリは非表示
- 検索クリアで全カテゴリ表示に復帰

### カテゴリ折りたたみ

- デフォルト: 「基本装飾」のみ展開、他は折りたたみ
- 展開状態は localStorage に保存（`zenwriter-mdref-collapsed`）

### 各項目のレイアウト

```
記法テキスト          ショートカット（あれば）
└ 簡潔な説明（1行）
```

- 記法テキスト: `<code>` で表示（等幅フォント）
- ショートカット: `<kbd>` で表示、右寄せ
- 説明: 控えめなフォントサイズ（0.85em）

### データ構造

```javascript
var MARKDOWN_REFERENCE = [
  {
    id: 'basic',
    title: '基本装飾',
    defaultOpen: true,
    items: [
      { syntax: '**太字**', shortcut: 'Ctrl+B', desc: '選択テキストを太字にする' },
      { syntax: '*斜体*', shortcut: 'Ctrl+I', desc: '選択テキストを斜体にする' },
      ...
    ]
  },
  ...
];
```

### 登録

```javascript
ZWGadgets.register('MarkdownReference', function(el) { ... }, {
  groups: ['assist'],
  title: 'Markdownリファレンス',
  description: 'Markdown記法・ショートカット・アプリ拡張の参照'
});
```

## 依存関係

- 前提: なし（独立したリファレンスコンテンツ）
- 関連: SP-055（WYSIWYG対応表）, SP-059（ルビ記法）, SP-060（装飾プリセット記法）

## 受け入れ基準

1. サイドバーの assist グループに「Markdownリファレンス」ガジェットが表示される
2. 全8カテゴリが折りたたみ可能で、検索フィルタが機能する
3. 各項目に記法・ショートカット（該当時）・説明が表示される
4. アプリ固有拡張（wikilink, ルビ等）がリファレンスに含まれる
