# SP-058 見出しタイポグラフィ仕様（Draft v0.1）

## 目的
既存の `--heading-h1...h6-*` 変数を UI から操作可能にし、章タイトルや節構造の可読性・演出力を高める。

## 背景（現状）
- H1-H6 の変数は既に定義済み（`css/style.css`, `css/variables.css`）。
- ただし編集UIは未提供で、実質固定値運用。
- 印刷スタイル（`css/print.css`）でも同変数を使用しており、編集時と出力時の統合余地が大きい。

## 仕様候補
- H1-H6 単位で以下を編集可能化:
  - `font-size`
  - `font-weight`
  - `line-height`
  - `margin-top / margin-bottom`
  - `letter-spacing`（新規）
- 「章扉」「本文重視」などの見出しプリセットを用意
- Sections Navigation（`SP-052`）と見出しレベル定義の整合ルールを文書化

## 優先順位
- 優先度: `P1`（高）
- 理由: 長編執筆で見出しの情報設計品質が直接 UX に効く。既存変数があり実装コストが低い。

## 依存関係
- 前提: `SP-054`（Typography 設定責務の明確化）
- 連携: `SP-052`（見出しナビ）, `SP-055`（WYSIWYG block editing）
- 出力連携: `print.css` 反映ルール

## 受け入れ基準
1. H1-H6 の調整値が編集画面と印刷プレビューで一致する。
2. 見出しプリセット適用後、個別微調整が可能。
3. 見出し設定変更が本文フォント設定を破壊しない。
4. 設定 JSON に未設定レベルがあっても既定値フォールバックで崩れない。

## 実装リスク（CTO観点）
- 6レベル全項目のUIは操作負荷が高く、初期版で過剰設計になりやすい。
- Preview/Print/Embed でスタイル適用順序がずれると表示不一致が発生。
- WYSIWYG の `h1-h3` 制限（`SP-055`）との差分が UX 混乱を招く可能性。

## 実装メモ
- 初期は H1-H3 を詳細、H4-H6 は簡易設定に制限する段階導入を推奨。
- CSS 変数の命名は既存 `--heading-h*-*` を踏襲し、追加分だけ定義。
- E2E は見出しサイズ反映とリロード復元を最小セット化。
