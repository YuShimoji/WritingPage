# 拡張スクロール機能 仕様書（Draft v0.1）

## 概要

本仕様は、Zen Writer の編集体験を「固定幅テキストエリア」から「広いキャンバス上をドラッグで見渡せる編集空間」へ拡張する。  
狙いは、クラフトテーブルを掴んで移動するような操作感で、テキストを上下左右に連続配置できるノート体験を提供すること。

---

## 背景課題

- 現行は `textarea` / `contenteditable` とも縦スクロール中心で、横方向の文脈展開が難しい
- 画面が広いタブレットでも、編集領域の自由度が活かしきれていない
- ズーム時に「本文文字サイズ」と「キャンバスの見え方」が分離されておらず、将来のリッチテキスト強化との整合が必要

---

## 目的

1. 余白を含めてエディタ空間を縦横にパン（ドラッグ移動）できること
2. 既存本文の上下左右に続きや補助メモを書けること
3. ズームしても本文の実効文字サイズを固定できること（可読性優先）
4. Editorの回転・拡大縮小・移動を同一モデルで扱えること

---

## 非スコープ（Phase 1）

- 複数ユーザー同時編集
- 無限精度のベクター編集機能
- 3D回転・遠近変形
- 自動レイアウト（自動整列/自動配線）

---

## 用語定義

- `Canvas Space`: ワールド座標系。ノード（文書ブロック）が配置される論理平面
- `Viewport`: 画面上の見えている領域
- `Pan`: Viewportの平行移動
- `Zoom`: Viewport倍率変更
- `Rotate`: 編集平面の回転（2D）
- `Text Node`: Markdown本文を保持する編集ブロック

---

## 機能仕様

## 1. 編集空間モデル

- 新規レイヤ `Editor Canvas Mode` を追加する
- 編集対象は単一本文を複数 `Text Node` に分割配置できる（Phase 2以降）
- 初期状態では中央に `Primary Text Node` を1つ配置
- ノード作成は以下で可能（Phase 2以降）:
  - ノード端の「+」ハンドル
  - ショートカット（例: `Alt+Drag` で複製）
  - コマンドパレット（`ノード作成: 上/下/左/右`）

## 2. パン（ドラッグ移動）

- 空白領域ドラッグでキャンバスを移動
- 推奨操作:
  - `Space + Drag`: 一時パン
  - `中ボタンドラッグ`: パン
  - タッチ: 2本指パン
- 慣性スクロールはタブレット優先で有効（設定でOFF可）

## 3. ズーム

- `Ctrl/Cmd + マウスホイール` でズーム
- UIボタン（`+`, `-`, `100%`）を提供
- ズーム範囲（推奨）: `25%` 〜 `400%`
- `fit-to-selection` / `fit-to-content` をコマンドとして提供

## 4. 文字サイズ固定（リッチテキスト連携）

- キャンバスズームは「座標系とボックス描画」に適用
- 本文の見かけ文字サイズは設定により固定可能:
  - `fixedTextScale: true`（既定推奨）
- 実現方針:
  - ノード外枠はズーム
  - ノード内テキストは逆スケール補正で視認サイズを維持
- `docs/specs/spec-richtext-enhancement.md` のCommand/Bridge方針に従い、WYSIWYGでも同じ見え方を維持

## 5. 回転

- Editor全体に回転角を適用できる（推奨範囲: `-45deg`〜`45deg`）
- 入力手段:
  - 回転スライダー
  - `Shift + ホイール`（設定ON時）
  - ノード単位回転（Phase 4候補）
- 回転中でもカーソル入力位置が破綻しないこと

## 6. 既存モードとの整合

- 既存 `textarea` / `WYSIWYG` モードを廃止せず共存
- モード構成:
  - `Classic Mode`（既存）
  - `Canvas Mode`（新規）
- 文書保存時、Canvas Modeでも本文正本はMarkdownを維持

---

## データ仕様

```json
{
  "editor": {
    "canvas": {
      "enabled": true,
      "view": {
        "panX": 0,
        "panY": 0,
        "zoom": 1,
        "rotation": 0,
        "fixedTextScale": true
      },
      "nodes": [
        {
          "id": "node-main",
          "x": 0,
          "y": 0,
          "w": 900,
          "h": 1200,
          "rotation": 0,
          "contentRef": "doc:main"
        }
      ]
    }
  }
}
```

### バリデーション

- `zoom`: `0.25..4.0` に clamp
- `rotation`: `-45..45` に clamp
- ノードサイズ最小: `w>=280`, `h>=160`
- ノード座標は数値以外を破棄し既定値へフォールバック

---

## UI仕様

## 1. キャンバスHUD

- 右下固定の最小HUD:
  - `Zoom +` / `Zoom -`
  - `100%`
  - `Rotate Reset`
  - `Mini-map` トグル

## 2. ミニマップ（推奨）

- 画面右上に小さな全体図を表示可能
- 現在Viewportを矩形で表示し、クリックでジャンプ
- 長文時の位置ロストを防止

## 3. モード切替

- 既存トグル群に `Canvas Mode` を追加
- 切替時は確認なしで即時反映（Undo可能）

---

## アクセシビリティ

- キーボードのみでパン・ズーム・ノード選択が可能
- `prefers-reduced-motion` 時は慣性や補間アニメを抑制
- スクリーンリーダー向けに「現在ノードID」「ズーム率」「回転角」を通知可能にする

---

## 受け入れ基準（Acceptance Criteria）

1. Canvas Modeで空白ドラッグによるパンが機能する
2. ズーム率変更後も、`fixedTextScale=true` なら本文文字の見かけサイズが維持される
3. テキストノードを上下左右に追加し、各ノードで入力・保存・再読み込みができる
4. 回転を適用してもテキスト入力とカーソル移動が破綻しない
5. Classic Modeへ戻しても既存文書互換が壊れない
6. 主要E2E（スクロール回帰、WYSIWYG、保存）で重大回帰がない

---

## 段階導入（推奨）

| Phase | 内容 | 判定 |
|------|------|------|
| Phase 1 | Canvas Viewport（pan/zoom）導入、単一ノード編集 | 必須 |
| Phase 2 | ノード増設・方向追加UI・ミニマップ導入 | 必須 |
| Phase 3 | 回転導入、fixedTextScaleとWYSIWYG同期 | 必須 |
| Phase 4 | 慣性/スナップ/ショートカット最適化 | 推奨 |

---

## リスクと対策

- リスク: CSS transform起因で入力キャレット座標がズレる
  - 対策: テキストレイヤーと装飾レイヤーを分離し、入力面は非回転補正レイヤで扱う
- リスク: 既存Overlay画像座標との不整合
  - 対策: 画像OverlayもCanvas座標系へ変換するブリッジを追加
- リスク: パフォーマンス低下（大型文書）
  - 対策: ノード単位で仮想化・表示領域外更新抑制

---

## 決定事項（2026-03-10）

- D1. 初期リリースは単一ノードを正式対応とし、複数ノードは Phase 2 で導入する
- D2. 回転はEditor全体回転のみを先行し、ノード個別回転は後続フェーズで扱う
- D3. タッチ既定操作は `1本指=編集` / `2本指=パン・ズーム` とする
