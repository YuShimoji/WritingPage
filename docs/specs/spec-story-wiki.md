# Story Wiki 仕様書

## 概要

物語の世界観を構築・管理するための用語集Wikiシステム。
ファンダム型の用語集Wikiを自動生成し、執筆を補助・盛り上げる。

- ガジェット名: `StoryWiki` (新規設計、レガシー2ガジェットを置換)
- 置換対象: `wiki.js` (Wiki) + `gadgets-wiki.js` (StoryWiki)

## Phase 構成

| Phase | 内容 | 状態 |
|-------|------|------|
| Phase 1 | 基本機能 (CRUD / ツリー / カテゴリ / 検索 / エディタ連携) | done |
| Phase 2 | AI生成 / グラフビュー / 高度な自動検出 | todo |

## Phase 1 仕様

### 1. データモデル

```javascript
WikiEntry = {
  id: string,           // UUID
  title: string,         // 用語名 (一意)
  category: string,      // カテゴリID
  aliases: string[],     // 別名・別表記 (自動検出/ハイライト対象)
  content: string,       // 本文 (Markdown)
  tags: string[],        // タグ (自由入力)
  relatedIds: string[],  // 関連記事のID
  createdAt: number,     // 作成日時 (timestamp)
  updatedAt: number,     // 更新日時 (timestamp)
  source: 'manual' | 'auto' // 作成元 (手動 or 自動検出)
}
```

### 2. カテゴリシステム

#### プリセットカテゴリ (7種)

| ID | 表示名 | 説明 |
|----|--------|------|
| `character` | キャラクター | 登場人物 |
| `location` | 場所 | 地名・建物・世界 |
| `item` | アイテム | 道具・武器・重要物 |
| `organization` | 組織 | 団体・国家・勢力 |
| `term` | 用語 | 作品固有の用語・言葉 |
| `event` | イベント | 事件・出来事・歴史的事象 |
| `concept` | 概念 | 法則・思想・体系 |

#### カスタムカテゴリ

ユーザーが任意のカテゴリを追加可能。

```javascript
WikiCategory = {
  id: string,       // ユーザー入力 or 自動生成
  label: string,    // 表示名
  icon: string,     // アイコン (lucide icon名)
  isPreset: boolean // プリセットかカスタムか
}
```

### 3. UI構成

#### 3.1 サイドバー簡易表示

サイドバーガジェットとしてコンパクトに表示。

```
┌─────────────────────┐
│ [検索バー        🔍] │
├─────────────────────┤
│ キャラクター    (12) │
│ 場所            (8) │
│ アイテム        (5) │
│ 組織            (3) │
│ 用語           (15) │
│ イベント        (4) │
│ 概念            (2) │
├─────────────────────┤
│ [+ 新規作成] [展開⤢] │
└─────────────────────┘
```

- 検索バー: インクリメンタル検索 (タイトル / 別名 / タグ / 本文)
- カテゴリ行クリック: そのカテゴリの記事リストをインライン展開
- `展開` ボタン: 全画面ペインに切り替え

#### 3.2 全画面ペイン (2ペイン構成)

```
┌──────────────┬───────────────────────────┐
│ [検索バー  🔍]│ タイトル                   │
│              │ カテゴリ: [キャラクター]     │
│ ▼ キャラクター│ 別名: 勇者, Hero           │
│   ├ 太郎     │ タグ: #主人公 #勇者         │
│   ├ 花子     │─────────────────────────── │
│   └ 魔王     │ ## 概要                    │
│ ▼ 場所      │ 本作の主人公。...           │
│   ├ 王都     │                           │
│   └ 魔王城   │ ## 経歴                    │
│ ▶ アイテム   │ 幼少期に...                │
│ ▶ 組織      │                           │
│ ▶ 用語      │ ## 関連項目                │
│ ▶ イベント   │ [[花子]] [[王都]]           │
│ ▶ 概念      │                           │
│              │                           │
│ [+ 新規作成] │ [編集] [削除]              │
└──────────────┴───────────────────────────┘
```

- 左ペイン: カテゴリ別ツリー (折りたたみ可能)
- 右ペイン: 記事詳細 (Markdownレンダリング)
- 編集モード: 右ペインがMarkdownエディタに切り替わる

### 4. 自動検出 (用語候補の提案)

#### トリガー

1. **保存時自動検出**: ドキュメント保存時に未登録の固有名詞候補を抽出
2. **手動スキャン**: ボタン押下で現在のテキストをスキャン

#### 検出ロジック

- カタカナ連続 (2文字以上)
- 漢字連続 (2文字以上) のうち一般的でないもの
- 大文字始まりの英単語
- 既存Wiki記事の aliases とのマッチング (除外用)
- 頻出語の除外リスト (助詞、一般名詞等)

#### 提案UI

```
┌─────────────────────────────────┐
│ 新しい用語候補が見つかりました     │
│                                 │
│ ☑ アルファガルド  → [場所  ▼]    │
│ ☑ セリーヌ       → [キャラ ▼]   │
│ ☐ エルドリッジ   → [組織  ▼]    │
│ ☐ 魔素          → [用語  ▼]    │
│                                 │
│ [選択した候補を登録] [すべて無視]  │
└─────────────────────────────────┘
```

- 各候補のカテゴリをドロップダウンで選択可能
- チェックで選択/解除
- 登録すると WikiEntry (source: 'auto') として追加
- 無視した用語は除外リストに追加

#### オン/オフ設定

- 設定でオン/オフ切り替え可能
- デフォルト: オン

### 5. エディタ連携

#### 5.1 `[[wikilink]]` 構文

- `[[用語名]]` で Wiki記事へのリンクを作成
- `[[用語名|表示テキスト]]` で表示テキストを変更可能
- WYSIWYGモード: リンクとしてレンダリング、クリックでWiki記事を開く
- プレーンテキストモード: 構文がハイライトされる
- 存在しない記事へのリンク: `is-broken` クラスで視覚的に区別

#### 5.2 登録済み用語の自動ハイライト

- エディタ本文中の登録済みWiki用語を自動的にハイライト表示
- ホバーでツールチップ (用語名 + カテゴリ + 本文冒頭)
- クリックでWiki記事を開く
- aliases も検出対象
- パフォーマンス: デバウンス処理 (入力停止後 500ms)

### 6. ストレージ

- Phase 1: localStorage (既存 storage.js の Wiki API を継承・拡張)
- ストレージキー: `zenWriter_story_wiki` (新キー)
- 旧データ (`zenWriter_wiki_pages`) からの自動マイグレーション

### 7. レガシー処理

- `wiki.js` (Wiki ガジェット): 削除
- `gadgets-wiki.js` (StoryWiki ガジェット): 削除
- `link-graph.js`: Phase 2 まで保持 (グラフビュー用)
- CSS: `.wiki-*` クラスは新UIに合わせて再設計
- E2E: `wiki.spec.js` / `wikilinks.spec.js` を新仕様に合わせて書き直し
