# メインハブ（ツールパネル）UI 整理スライス（バックログ）

## 現状（事実）

- `#main-hub-panel` の「クイックツール」タブは、`#main-hub-toolbar-proxy` にサイドバー操作の **プロキシボタン**を並べた最小実装（`index.html` + `main-hub-panel.js` の `data-proxy-click`）。
- タブが多く、クイックツール内に **HUD 制御・本文サイズ** なども同居しており、解像度を上げていない状態で「モック的にごちゃごちゃ」に見えやすい。

## ゴール（別タスクでよい範囲）

1. **情報設計**: クイックツール＝本当に「頻出の 3〜5 操作」に絞る。検索・装飾等はタブ専用へ誘導のみ。
2. **ビジュアル**: `panel-row--wrap` の小ボタン群を、既存の `sidebar-chrome-toolbar` と同格のトークン（間隔・最小タッチ幅・アイコン優先）に揃える。
3. **HUD ブロック**: 一般用と開発者用の切り分け、または設定モーダルへの移管を検討。
4. **i18n**: プレースホルダ文言の `data-i18n` 統一。

## 非ゴール

- メインハブの機能削除そのもの（プロキシ経路は執筆導線の安全網のため、整理は段階的に）。

## 受け入れ条件（草案）

- 1280px 幅でクイックツールタブが **折り返し崩れなく** 読める。
- 主要操作が **キーボードのみ** で辿れる（既存ショートカット表との整合）。
