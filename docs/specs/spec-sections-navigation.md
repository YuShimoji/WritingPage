# セクションツリー/話ナビゲーション 仕様書

## 概要

部・章・節（および話数）を「挿入するだけ」で終わらせず、  
左サイドでツリー管理し、選択した話を編集の主対象として扱える導線を追加する。

関連仕様: `docs/specs/spec-writing-focus-sidebar.md`（執筆集中表示と章/シーン遷移）

合わせて、エディタ最下部に「前の話 / 次の話 / 見出しへ移動」ナビを常設し、
長編執筆時の移動コストを下げる。

---

## 背景課題

- アウトラインは見出しテンプレート挿入機能はあるが、挿入後の構造を一覧管理できない
- 「今どの話を編集中か」が分かりにくく、話間移動が遅い
- 左サイドの主要導線がガジェット中心で、セクション管理が最上位にない
- 話の前後移動を毎回スクロールや検索で行う必要がある

---

## 目的

- セクション（話）をツリーで可視化する
- 話を選択したら、その話へ即ジャンプして編集できる
- 左サイド最上段に「セクション」カテゴリを新設する
- エディタ下部から前後話へワンクリック移動できる

## 機能仕様

### 1. サイドバーに「セクション」カテゴリを新設

#### 1.1 表示順

- アコーディオン順は以下に固定する
  1. `sections`（新規）
  2. `structure`
  3. `edit`
  4. `theme`
  5. `assist`
  6. `advanced`

#### 1.2 初期状態

- `sections` は初期展開
- `SectionsNavigator` は初期ロードアウトで最上位

### 2. SectionsNavigator（左サイドツリー）

#### 2.1 見出し抽出ルール

- Markdown見出し行 `^(#{1,6})\s+(.+)$` を対象
- 各見出しノードに以下を持つ
  - `id`（内部ID）
  - `level`（1-6）
  - `title`
  - `startOffset`
  - `endOffset`（次見出し開始位置、なければ文末）
- 1文書内でツリー表示（インデントで階層化）

#### 2.2 基本操作

- ノードクリック:
  - エディタを該当話位置へスクロール
  - カーソルを見出し先頭へ移動
  - そのまま編集可能
- `+ 新規セクション`:
  - 現在カーソル位置に既定見出し（例: `### 第N話 タイトル`）を挿入
  - `N` は既存話数から自動採番
- `更新`:
  - 本文を再解析してツリーを再構築

#### 2.3 アクティブ話の追従

- カーソル位置から現在話を判定し、ツリー側をハイライト
- 文書切替・保存・入力時に自動再同期

### 3. エディタ最下部の話ナビ

#### 3.1 常設ナビ

- エディタ下部に常設:
  - `前の話`
  - `次の話`
  - `見出しへ移動`
  - `<話タイトル>`

#### 3.2 動作

- `前の話`:
  - 現在話の1つ前へジャンプ（先頭時は無効）
- `次の話`:
  - 現在話の1つ後へジャンプ（末尾時は無効）
- `見出しへ移動`:
  - 左サイドを開き `sections` カテゴリを表示
  - 該当ノードへフォーカス
- 同期:
  - サイドバーで章/シーン移動した場合も `<話タイトル>` を即時反映
  - 下部ナビで前後移動した場合もサイドバーのアクティブ章/シーンを即時反映

### 4. 話選択時の編集挙動

- Phase 1（本仕様範囲）:
  - **全文表示のまま**選択話へジャンプし、そこで編集する
  - データ分割や部分バッファ化は行わない
- 将来拡張（Phase 2）:
  - 話単位のフォーカス編集モード（部分表示・部分保存）を検討

## 非機能要件

- 10,000行規模文書でも入力遅延を増やしすぎない
  - 再解析はデバウンス（目安 80-150ms）
- キーボード操作可能
  - ツリーノードは `button`
  - 下部ナビはTabで到達可能
- テーマ互換（ライト/ダーク）

---

## データ/イベント仕様

### データ

- 新規永続データは追加しない（Phase 1）
- 見出し構造は本文から都度再構築する派生データ

### 連携イベント

- 入力: `editor input`
- カーソル移動: `click / keyup / focus`
- 文書切替: `ZWDocumentsChanged`
- 保存: `zen-content-saved`
- 章/シーン位置同期:
  - 送信: `ZWWritingFocusLocationChanged`（サイドバー/下部ナビ双方で受信）
  - 受信: `ZWBottomNavNavigate`（`prevChapter` / `nextChapter` / `prevScene` / `nextScene` / `openSections`）

---

## 受け入れ基準（Acceptance Criteria）

1. 左サイド最上段に「セクション」カテゴリが表示される
2. 本文に見出しが2つ以上あるとツリー表示される
3. ツリーの項目クリックで該当話位置へ移動し、編集できる
4. `+ 新規セクション` で見出しが挿入され、ツリーに即反映される
5. エディタ下部に `前の話 / 次の話 / 見出しへ移動` が表示される
6. 前後話ボタンで順方向/逆方向に話移動できる
7. 文書切替後もツリーと下部ナビが新文書内容に追従する

---

## 非スコープ

- 話ごとの独立ドキュメント化
- 話単位の差分履歴/ロック
- AIによる見出し自動生成
- 既存アウトラインプリセットUI自体の全面置換

### 技術的な注意点

- **WYSIWYGモード対応**: `getEditorText()` で `richTextEditor.syncToMarkdown()` を呼び出してからtextarea値を読む
- **イベントリスナー**: textarea `input` + WYSIWYG `input` + `document` `zen-content-saved` + `window` `ZWDocumentsChanged`
- **デバウンス**: 120ms でheading再解析（`scheduleRender`）
- **スクリプト読み込み順**: `gadgets-builtin.js`（`window.ZWGadgets` 生成）の後に配置が必須

### E2Eテスト（5件）

1. セクションカテゴリがサイドバーに存在し最上段にある
2. 見出しツリーが正しくレンダリングされる（4ノード、タイトル一致）
3. ツリーノードクリックでエディタカーソルがジャンプする
4. エディタ下部ナビ（prev/next/title）が表示される
5. 下部ナビにアクティブセクション名が表示される
