# 読者プレビューモード仕様書 (SP-078)

## 概要

執筆した小説を「読者が読む体験」としてプレビューする第4のUIモード。
既存のNormal/Focus/Blankモードに加えて、編集UIを完全に隠し、
読者向けの閲覧体験を提供する。

---

## 背景課題

- 執筆ワークフローの「構想→章立て→執筆→校正→**出力/共有**」のうち、出力/共有が完全に欠落
- 現在のプレビューパネルは編集UIと併存しており、「読者として読む」体験を再現できない
- draft/hidden章のフィルタリング、章末ナビ、目次は既に実装済みだが、読者向けに統合されていない
- 「書いたものがどう見えるか」を確認するフローがなく、執筆ツールとしてのループが閉じない

---

## 目的

- 読者が作品を読む体験をプレビューできるようにする
- 編集UIを完全に隠し、本文・目次・章ナビのみを表示する
- visible章のみを表示し、draft/hidden章を自動除外する
- chapter://リンクが読者向けに機能する（章間ジャンプ）
- 将来のHTML/EPUB出力の基盤となる読者向けレンダリングを確立する

---

## モード定義

### Reader モード（第4モード）

| 要素 | 表示 |
|------|------|
| サイドバー | 非表示 |
| ガジェット | 非表示 |
| ヘッダーツールバー | 非表示（上端ホバーで「編集に戻る」FABのみ） |
| フローティング装飾バー | 非表示 |
| ステータスバー | 非表示 |
| エディタ（textarea/WYSIWYG） | 非表示 |
| 読者プレビュー | **全画面表示** |

実装名称: `data-ui-mode="reader"`

---

## 機能仕様

### 1. 読者プレビューの構成

```
┌──────────────────────────────────────────────┐
│                [編集に戻る]     (上端ホバー)    │
├──────────────────────────────────────────────┤
│                                              │
│  作品タイトル                                 │
│  ────────────                                │
│                                              │
│  目次                                        │
│  1. 第1章: 始まり                             │
│  2. 第2章: 森の入口                           │
│  3. 第3章: 青い屋根                           │
│                                              │
│  ────────────────────────────────────────    │
│                                              │
│  第1章: 始まり                                │
│                                              │
│  本文が表示される...                          │
│                                              │
│  [← 前の章]    [目次]    [次の章 →]           │
│                                              │
│  ────────────────────────────────────────    │
│                                              │
│  第2章: 森の入口                              │
│  ...                                         │
│                                              │
└──────────────────────────────────────────────┘
```

### 2. コンテンツ生成ルール

- **visible章のみ**: `getVisibleChapters()` でフィルタリング
- **draft/hidden章**: 完全に除外（テキストもナビも非表示）
- **目次**: プレビュー先頭に自動生成（`injectToc()` 再利用）
- **章末ナビバー**: 各章末に自動挿入（`injectNavBars()` 再利用）
- **chapter://リンク**: アンカーリンクに変換（`convertForExport()` 再利用）
- **壊れリンク**: 読者モードでは非表示（editモード専用の警告）
- **装飾パイプライン**: editor-preview.js と同等の変換を適用
  - テキストボックス: `TextboxRichTextBridge.projectRenderedHtml()` で投影
  - フォント装飾: `processFontDecorations()` で変換
  - テキストアニメーション: `processTextAnimations()` で変換 (wave/sparkle/cosmic/fire/glitch)
  - `[[wikilink]]`: `<a class="wikilink">` に変換（存在チェック付き）
  - 傍点: `{kenten|text}` → `<span class="kenten">`
  - ルビ: `{漢字|かな}` → `<ruby>` / `|漢字《かな》` → `<ruby>`
- **選択肢グループ化**: `autoGroupChoices()` で連続する chapter-link を自動グループ化

### 3. スタイリング

- 最大幅: 720px（読みやすい行長）
- フォント: Visual Profile の読者向けプロファイル（未実装の場合は本文フォント継承）
- 背景: テーマ設定に従う（ダークモード対応）
- リンク色: テーマ変数 `--link-color`
- 印刷時: print.css を適用

### 4. モード切替

- **起動**: ツールバーの「読者プレビュー」ボタン（目のアイコン）
- **復帰**: 上端ホバーで「編集に戻る」FAB、またはEscキー
- **ショートカット**: Ctrl+Shift+R（フォールバック）

### 5. 読者プレビュー内のインタラクション

| 操作 | 動作 |
|------|------|
| 目次リンクをクリック | 該当章へスクロール |
| 章末ナビをクリック | 前/次の章へスクロール |
| chapter://リンクをクリック | 該当章へスクロール |
| テキスト選択 | 通常のテキスト選択（コピー可） |
| 右クリック | ブラウザ標準メニュー |
| スクロール | 通常スクロール |
| Esc | 編集モードに復帰 |

---

## ユーザー操作フロー

### フロー1: 読者プレビューで確認

1. 執筆中にツールバーの「読者プレビュー」ボタンをクリック
2. 全画面で読者向けレイアウトが表示される
3. 目次をクリックして各章を確認
4. 章末ナビで次の章へ移動
5. 確認が終わったらEscまたは「編集に戻る」で復帰

### フロー2: draft章の非表示確認

1. 一部の章をdraftに設定
2. 読者プレビューに切り替え
3. draft章が目次・本文・ナビから除外されていることを確認
4. 公開設定の調整後、再度プレビュー

## 成功状態

- ツールバーのボタンクリックで読者プレビューに遷移する
- 編集UIが完全に隠れ、読者向けレイアウトのみ表示される
- visible章のみが目次・本文・ナビに表示される
- chapter://リンクが章内アンカーとして機能する
- Escで編集モードに確実に復帰する
- ダークテーマでも読みやすい表示になる

## 実装計画

### Phase 1: 基本モード（MVP）

- `data-ui-mode="reader"` のCSS定義（全UI非表示 + 読者プレビューパネル全画面）
- 読者プレビューパネルの生成（全visible章を1つのHTMLに結合）
- 目次 + 章末ナビの注入
- モード切替ボタン（ツールバー） + Esc復帰
- 上端ホバーFAB（「編集に戻る」）

### Phase 2: スタイリング

- 読者向けタイポグラフィ（最大幅720px、行間・フォントサイズ調整）
- テーマ対応（全6テーマ）
- 印刷最適化

### Phase 3: 拡張

- [x] HTML出力（「HTMLとして保存」ボタン）
  - 読者プレビュー上部ホバーで「HTMLとして保存」ボタン表示
  - スタンドアロンHTML生成（テーマ色・フォント・タイポグラフィ埋め込み）
  - 章末ナビバーは出力から除外（編集ツール用）
  - Blobダウンロードでファイル保存
  - HUDフィードバック「HTMLを保存しました」
- [x] 読書進捗（プログレスバー + スクロール位置の記録・復元）
- [ ] Visual Profile 読者プロファイルとの連携

- 読者認証・アクセス制御
- 縦書き表示
