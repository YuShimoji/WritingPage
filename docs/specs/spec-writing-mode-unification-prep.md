# 執筆モード統合 事前整理（引き継ぎ用）

最終更新: 2026-04-08  
対象: WP-001（UI/UX 磨き）主導、WP-004（見え方整合）と競合しない範囲

## 目的

執筆時の混乱を減らすため、UI モードと編集面の責務を明確化し、次の実装プランで「何を固定し、何を保留するか」を共有可能にする。

---

## この時点の合意事項（固定）

1. 執筆の標準は `focus`。  
2. `normal` は必要時の編集/設定モード（補助）として扱う。  
3. Reader は UI モードとしては廃止し、再生オーバーレイで代替する。  
4. ヘルプは任意参照（執筆導線を汚さない）。  
5. 左サイドバーはライター機能を優先し、常設 UI を最小化する。

---

## 問題の構造（不安定に感じる理由）

### 1) モード差が見た目差に偏る
- `normal` / `focus` の責務が「表示 ON/OFF」に寄り、目的差が弱く見える。

### 2) UI モード軸と編集面軸が混在
- UI モード (`normal` / `focus`) と編集面（WYSIWYG / Markdown）が別軸なのに、同列の「モード」のように認知される。

### 3) 復帰導線の期待差
- 再生確認後に「どこへ戻るか（フォーカス・表示状態・編集面）」の期待が揺れる。

### 混在リスク
- 用語（Reader モード / 再生オーバーレイ / 読者プレビュー）が文書間で不一致になると、仕様理解が分岐する。
- 「正本」は定義されているが、語彙同期が遅れると実質的に SSOT が崩れる。

### このプロジェクトで合理的な SSOT 設計
- `INVARIANTS`: 破ってはいけない条件のみ
- `INTERACTION_NOTES`: 用語・状態モデル（説明）
- `CURRENT_STATE`: セッション事実・検証履歴
- `FEATURE_REGISTRY`: 機能 ID と実装参照
- 本ドキュメント: プラン前の意思決定メモ（固定事項と未決定事項の境界）

## 用語の固定（混同防止）

### UI モード
- `focus`: 執筆標準
- `normal`: 補助モード（設定・管理）

### 編集面（surface）
- WYSIWYG: 視覚編集の面（UI モードではない）
- Markdown ソース: 記法編集の面（UI モードではない）

### 再生オーバーレイ
- 読者視点確認の一時 UI
- UI モード自体は変えない

## 保存まわり（今回の重要論点）

### 背景
- ガジェット化を強めた結果、保存・補助・管理導線が散在しやすく、主要操作が埋もれる懸念がある。

### 決めるべき仕様（未決定）
1. 一時保存だけで十分か、手動保存ボタンを常設するか
2. 手動保存を置く場合の配置（常設枠 / ドック / コマンド）
3. 自動保存の調整 UI（遅延、ON/OFF、通知）をガジェットに寄せる範囲
4. 保存状態の可視化（保存済み/未保存/保存中）をどこで見せるか

### 決定（session 71）
1. 既定運用は **自動保存中心** を維持し、手動保存の常設ボタンは追加しない。
2. 手動保存は **コマンドパレット / ショートカット / ドキュメント系ガジェット** を主導線とする（常設 chrome には増設しない）。
3. 自動保存の詳細調整（遅延・ON/OFF・通知）は設定/ガジェット側に集約し、執筆面の常設 UI には露出を増やさない。
4. 保存状態の可視化は「見つかるが邪魔でない」を優先し、最小表示に留める（派手な常時表示は採用しない）。

### 採用理由（session 71）
- `focus` 標準運用で執筆導線の認知負荷を増やさないため。
- 既存の保存導線（Ctrl/Cmd+S・コマンド・ドキュメント系ガジェット）で手動保存の到達性は確保できているため。
- 追加の常設ボタンは操作ミス低減よりも UI 密度上昇のデメリットが大きいと判断したため。

### 判断基準（推奨）
- 執筆中に高頻度で使うか
- キーボードのみで完結できるか
- 誤操作時に復旧しやすいか（安全性）
- UI 常設の認知負荷を増やさないか

## 一時 UI（フローティング/オーバーレイ）へ押し込む境界

### 押し込める対象
- 再生確認
- ヘルプ
- 低頻度設定
- 補助ガジェット

### 常設が望ましい対象
- 章一覧/章移動
- 最小限の保存状態表示
- 執筆中に連続で使う導線

---

## ガジェット戦略（暫定）

### 方針
- 「全部ガジェット化」は採らない。
- ライター常用導線は常設枠へ、低頻度機能はドック/折りたたみへ。

### 次スライスで詰める項目
1. 常設枠に残す最小セット（章、保存状態、再生導線）
2. ドックへ退避する候補（Wiki、詳細設定、補助）
3. ガジェットの表示優先度（初期表示/折りたたみ）

---

## 受け入れ基準（プラン設計時の評価軸）

1. 執筆開始時の迷いがない（`focus` が標準）
2. 章間移動がワンクリック
3. 再生確認後に編集面へ確実復帰
4. 保存導線が「見つかるが邪魔でない」
5. Electron とキーボード完結を維持
