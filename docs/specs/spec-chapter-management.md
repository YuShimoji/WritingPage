# チャプター管理再設計 仕様書 (SP-071)

## 概要

小説のセクション分け・章分けをNovlr風の2ペインUIに再設計する。
左パネルにチャプターリスト、右にエディタという構成で、
長編執筆時の章間移動と構造把握を直感的にする。

SP-070 (モードアーキテクチャ) の **Focus モード** における主要UIとして機能する。

---

## 背景課題

- 現在の章構造は見出しテンプレート挿入 + SectionsNavigator ガジェットで管理しているが、
  エディタ内に全章が縦に並ぶため長編では管理しにくい
- 見出しレベルの深さと章構造の対応が書き手の裁量に依存しており、基準がない
- 章の追加・並び替え・削除がエディタ内のテキスト操作に依存している
- Focusモードで「タイトルと章構造だけ」を表示する受け皿がない

---

## 目的

- 章（チャプター）を独立した管理単位として扱えるようにする
- 左パネルで章を選択し、右パネルでその章のみを編集する構造にする
- 章の追加・並び替え・リネームをテキスト編集とは独立した操作として提供する
- 目次ページを自動生成できるようにする（SP-072 と連動）

---

## 構造モデル

### 階層

```
作品 (Document)
├── メタ情報 (タイトル、著者、概要)
├── 部 (Part) ← 任意。使わなくてもよい
│   ├── 章 (Chapter)
│   │   ├── 節 (Section) ← 任意。章内の見出しで表現
│   │   └── ...
│   └── ...
├── 章 (Chapter) ← 部なしで直接置くことも可
│   └── ...
└── 特殊ページ
    ├── 目次 (自動生成)
    ├── あとがき
    └── ユーザー定義ページ
```

### 章の実体

各章は以下を持つ:

| プロパティ | 型 | 説明 |
|-----------|------|------|
| id | string | 一意識別子 (UUID) |
| title | string | 章タイトル |
| parentId | string / null | 所属する部のID (部なしの場合 null) |
| order | number | 同一階層内での表示順 |
| content | string | 本文 (Markdown / WYSIWYG HTML) |
| metadata | object | 文字数、最終更新日時、メモ等 |
| visibility | string | "visible" / "draft" / "hidden" (SP-072 連動) |

## UI仕様

### Focus モード: チャプターリスト

左パネルにチャプターリストを表示する。

#### レイアウト

```
┌─────────────────┬──────────────────────────────────┐
│  [作品タイトル]   │                                  │
│  ────────────    │     エディタ                      │
│  ▼ 第一部        │     (選択中の章の本文)             │
│    1. 始まり     │                                  │
│    2. 旅立ち  ← │                                  │
│  ▼ 第二部        │                                  │
│    3. 再会       │                                  │
│    4. 決断       │                                  │
│  ────────────    │                                  │
│  + 新しい章      │                                  │
│                  │                                  │
│  [目次] [設定]   │              [ステータスバー]      │
└─────────────────┴──────────────────────────────────┘
```

#### チャプターリストの操作

| 操作 | 動作 |
|------|------|
| クリック | 章を選択し、右パネルにその章の本文を表示 |
| ダブルクリック | 章タイトルのインライン編集 |
| 右クリック | コンテキストメニュー（リネーム / 複製 / 削除 / 上へ / 下へ） |
| ドラッグ&ドロップ | 章の並び替え（部をまたいだ移動も可） |
| `+ 新しい章` | 末尾に新しい章を追加 |
| `Ctrl+N` | 現在の章の下に新しい章を挿入 |

#### 視覚的フィードバック

- 選択中の章: アクセントカラーの背景
- ドラフト / 非表示の章: テキストを薄くし、アイコンで区別
- 文字数: 各章の右端に薄い文字で表示（設定でON/OFF可）

### Editor モードとの関係

- Editor モードでは、現行の `SectionsNavigator` ガジェットが引き続き機能
- `SectionsNavigator` と `ChapterList` はデータソースを共有
- Editor モードではエディタ内に全章が見出しとして表示される（現行動作を維持）
- Focus モードでは章ごとに分離された編集ビューに切り替わる

## データ保存

### 現行からの移行

現在は1つのドキュメント内に全章が見出しで記述されている。
新しいチャプターモデルへの段階的移行:

1. **Phase 1 (互換)**: 既存の見出しベースの章を自動検出し、チャプターリストに表示。
   データ保存は現行形式のまま（1ドキュメント、見出しで分割）。**done**
2. **Phase 2 (分離)**: 章ごとに独立した保存単位に分離。
   インポート/エクスポート時に結合。**done**

### Phase 1 の保存形式

```
// localStorage key: "zenwriter-chapter-config"
{
  "documentId": "...",
  "parts": [
    { "id": "...", "title": "第一部", "order": 0 }
  ],
  "chapters": [
    {
      "id": "...",
      "title": "始まり",
      "parentId": "part-id or null",
      "order": 0,
      "headingOffset": 0,
      "visibility": "visible"
    }
  ],
  "specialPages": [
    { "id": "...", "type": "toc", "title": "目次" }
  ]
}
```

Phase 1 では `headingOffset` でドキュメント内の見出し位置を参照し、
エディタ表示時にその範囲だけを表示する。

### Phase 2 の保存形式

章は既存の `documents` IDB ストアに `type: 'chapter'` として保存する。

```javascript
{
  id: 'ch_1710000000000_abc123',
  type: 'chapter',
  parentId: 'doc_...',        // 所属ドキュメントID
  name: '第一章 始まり',       // 章タイトル
  content: '本文テキスト...',  // 章の本文 (Markdown)
  order: 0,                   // 表示順
  level: 2,                   // 見出しレベル (H1-H6)
  visibility: 'visible',      // visible / draft / hidden
  createdAt: 1710000000000,
  updatedAt: 1710000000000
}
```

- ドキュメントに `chapterMode: true` フラグで opt-in
- Focus モードで章を選択 → その章の content のみを editor に表示
- Normal モードでは全章を結合した全文テキストを表示
- モード切替時に双方向同期 (assembleFullText / splitIntoChapters)
- 移行は Focus モードのチャプターリスト下部「章モードに変換」ボタンで実行

---

## 目次ページ

- チャプターリストから自動生成される目次ページを提供
- 目次の各項目はクリックで該当章にジャンプ（エディタ内ナビゲーション）
- 出力時（HTML/印刷）にも目次として挿入可能
- 目次の表示/非表示は作品設定で制御

---

## 成功状態

- Focusモードで左パネルにチャプターリストが表示され、章をクリックすると右パネルに本文が出る
- 章の追加・リネーム・並び替えが直感的にできる
- 既存の見出しベースのドキュメントを開いたとき、自動的にチャプターリストが構築される
- EditorモードとFocusモードで同じドキュメントを矛盾なく扱える
