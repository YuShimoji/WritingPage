# 仕様: コンテキストベースツールバー

> Status: IMPLEMENTED / Phase 1-3 完了
> 対象バージョン: v0.3.29

## 背景と課題

v0.3.28でミニマルツールバーに移行した結果、`.toolbar-actions`が`display: none`で完全に非表示になり、以下のボタンにユーザーがアクセスできない状態になっている:

- 表示モード切替 (通常/フォーカス/ブランク)
- フォント装飾パネル
- テキストアニメーションパネル
- プレビュー/分割ビュー
- WYSIWYG切替
- 設定/ヘルプ/テーマ切替/全画面

E2Eテストは`showFullToolbar()`ヘルパーで回避しているが、実ユーザーには到達手段がない。

## 設計方針: コンテキスト表示

「全部並べる」旧デザインでも「全部隠す」現状でもなく、**ユーザーの操作文脈に応じて必要なツールだけを表示する**。

## 構成

### Layer 1: ミニマルヘッダー (常時表示)

```
[≡] [文字数] ───────────────────── [表示モード] [全画面]
```

| 要素 | 説明 |
|------|------|
| サイドバートグル | 既存のまま |
| 文字数カウント | 既存のまま |
| 表示モードスイッチ | select → アイコンボタン3個 (通常/フォーカス/ブランク) |
| 全画面ボタン | 既存 `#fullscreen` を常時表示に昇格 |

**除外**: 設定/ヘルプ/テーマ切替 → サイドバーに移動済み

### Layer 2: フローティング装飾バー (テキスト選択時に表示)

既存の `#wysiwyg-toolbar` を拡張し、テキスト選択時に選択範囲の近くにフローティング表示する。

```
┌────────────────────────────────────────────────┐
│ B  I  U  S  │  link  │  装飾▼  │  アニメ▼  │
└────────────────────────────────────────────────┘
```

| 要素 | 動作 |
|------|------|
| B/I/U/S | execCommand (太字/斜体/下線/取り消し線) |
| link | リンク挿入 |
| 装飾 ▼ | ドロップダウン: smallcaps, shadow, glow, wide, narrow, outline等 |
| アニメ ▼ | ドロップダウン: fade, slide, typewriter等 |

**表示条件**: WYSIWYGモード中にテキストが選択されたとき
**位置**: 選択範囲の上 (上に余白がなければ下)
**非表示**: 選択解除時またはクリック外

テキストモード時は従来通り `[tag]...[/tag]` を挿入。フローティングバーは表示しない。

### Layer 3: サイドバー (既存アコーディオン)

以下の機能はサイドバーのアコーディオンカテゴリから操作する:

| 機能 | カテゴリ |
|------|----------|
| テーマ選択/切替 | theme |
| プレビュー/分割ビュー | edit |
| WYSIWYG切替 | edit |
| 設定 | advanced |
| ヘルプ | advanced |

**注意**: プレビュー/分割ビュー/WYSIWYG切替は現在ツールバーにしかない。サイドバーのeditカテゴリにボタンまたはトグルとして追加する必要がある。

### Layer 4: キーボードショートカット (常時有効)

| ショートカット | 機能 |
|---------------|------|
| Ctrl+B | 太字 |
| Ctrl+I | 斜体 |
| Ctrl+U | 下線 |
| Ctrl+Shift+P | プレビュー切替 |
| F11 | 全画面 |
| Ctrl+/ | ヘルプ |

## 変更対象ファイル

1. **index.html** -- toolbar-actions内の再構成 (不要ボタン除去、モードスイッチ追加)
2. **css/style.css** -- ミニマルツールバーCSS修正、フローティングバーCSS
3. **js/editor-wysiwyg.js** -- WYSIWYGツールバーのフローティング化 + 装飾ドロップダウン
4. **js/gadgets-init.js** or **sidebar-manager.js** -- editカテゴリにプレビュー/WYSIWYG切替追加

## 段階実装

### Phase 1: ミニマルヘッダー復活
- 表示モードスイッチとフルスクリーンを`.toolbar-actions`から取り出して常時表示
- `.toolbar-actions`の残りは引き続き非表示

### Phase 2: フローティング装飾バー
- 既存`#wysiwyg-toolbar`をフローティング化
- 装飾/アニメーションドロップダウン追加

### Phase 3: サイドバー統合
- editカテゴリにプレビュー/分割ビュー/WYSIWYGトグルを追加
- advancedカテゴリにヘルプボタン追加

### Phase 4: エッジホバーUI (IMPLEMENTED)

スクリーンエッジにカーソルを近づけると、隠れたUIが一時的にスライドイン表示される。

| エッジ | UI | 動作 |
|--------|-----|------|
| 上端 (18px) | ツールバー | 上からスライドイン |
| 左端 (18px) | サイドバー | 左からスライドイン |

**タイミング**:
- 滞在時間 280ms でUI表示開始
- UI/エッジゾーンから離脱後 500ms で非表示
- UI上にカーソルがある間は維持

**対象モード**: ブランクモード/ツールバー非表示時に特に有効。通常モードでUIが既に表示されている場合は発動しない。

**実装ファイル**: `js/edge-hover.js`, `css/style.css` (エッジホバーセクション)

**CSS方式変更**: ツールバー非表示を `display: none` から `transform: translateY(-100%)` に変更。エッジホバーのスライドインアニメーションとの互換性を確保。

## 移行戦略

- `.toolbar-actions`は最終的に削除 (全機能が別の場所に移動後)
- E2Eの`showFullToolbar()`ヘルパーは段階的に不要になる
- 既存のキーボードショートカットは維持
