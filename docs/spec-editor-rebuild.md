# SP-081 エディタ体験再構築

> 作成: 2026-03-29
> ステータス: in-progress

## 背景

現在のエディタは以下の問題を抱えている:

1. **モード切替の状態管理が壊れている**: Focus/Normal/Blank/Reader の4UIモード + WYSIWYG/Textarea のエディタモードが混在し、切替時にサイドバーが執筆エリアに被さる、ツールバーが閉じない等の問題が発生
2. **自動挿入テキストの増殖**: 「新しい章」追加時に挿入されるタイトルが重複・増殖する。モック的な仮実装のまま
3. **「章モードに変換」の不明瞭さ**: heading-based (Phase 1) と independent chapters (Phase 2) の二重管理。変換/解除ボタンがバグを含んでいる
4. **フローティングツールバーの論理的不統一**: モードごとに異なる挙動で、大半が機能していない。状態管理が `data-visible` 属性 + JS プロパティ + CSS の三重管理
5. **エッジホバーのフェードインが予兆なし**: マウスを画面端に寄せるとUIが出現するが、何の前兆もなく分かりづらい

## 設計方針

### 削除するもの

- **「章モードに変換」「章モードを解除」ボタンと Phase 1 (heading-based) 章管理**: chapterMode を唯一の方式にする。二重管理を根絶
- **legacy heading-based 章検出 (chapter-model.js の heading-offset 方式)**: Phase 2 (independent chapters) に一本化。chapter-model.js は章名生成ユーティリティとして残す
- **renderMigrateButton() / renderRevertButton()**: 変換/解除UI全体を削除
- **auto-insert される「新しい章」タイトルの増殖ロジック**: 新規章は空コンテンツで作成し、章名のみ設定

### 修正するもの

#### M-1: モード切替時の状態管理

- UIモード切替 (setUIMode) 時に、サイドバー・ツールバー・フローティングツールバーの状態を確定的にリセット
- Focus モードに入る時: サイドバーを閉じる、メインツールバーを隠す、フローティングツールバーを隠す
- Normal モードに戻る時: サイドバー状態を復元、メインツールバーを表示、エッジホバー状態をクリア

#### M-2: フローティングツールバーの統一

- 表示/非表示の状態を `data-visible` 属性のみで管理 (`this._floatingVisible` プロパティを廃止)
- WYSIWYG モード以外では確実に非表示
- モード切替時に明示的に `_hideFloatingToolbar()` を呼ぶ

#### M-3: 新規章作成の簡素化

- 「新しい章」クリック時: 空コンテンツ + 「新しい章」名で章レコードを作成
- エディタへの自動テキスト挿入を行わない
- 作成後、章名のインライン編集を開始

#### M-4: エッジホバーのヒントテキスト

- Focus/Blank モードで、エッジ付近 (上部・左部) に半透明のヒントテキストを常時表示
- 上部: 「マウスを画面上部に移動するとツールバーが表示されます」
- 左部: 「マウスを画面左端に移動すると章パネルが表示されます」
- ヒントはエッジホバーが発火したら消える (初回表示後)
- localStorage で「表示済み」を記録し、2回表示されたら以後は非表示

#### M-5: chapterMode のデフォルト化

- 新規ドキュメントは常に chapterMode: true
- 既存の非 chapterMode ドキュメントは初回 focus モード遷移時に自動変換 (confirm なし)
- Phase 1 専用のコードパス (`refreshLegacyMode` 等) を削除

## 成功条件

1. モード切替 (Normal <-> Focus <-> Blank) でサイドバー・ツールバーが正しく表示/非表示される
2. 「新しい章」をクリックしても自動テキストが増殖しない
3. 「章モードに変換」ボタンが存在しない
4. フローティングツールバーが WYSIWYG モード + テキスト選択時のみ表示される
5. Focus/Blank モードでエッジホバーのヒントが見える
6. E2E テストが回帰なしで通過する

## 実装順序

1. レガシー削除 (M-5 + 「章モードに変換」関連)
2. モード切替修正 (M-1)
3. フローティングツールバー修正 (M-2)
4. 章作成の簡素化 (M-3)
5. エッジホバーヒント (M-4)
