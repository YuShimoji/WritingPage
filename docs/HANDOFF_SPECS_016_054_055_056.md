# Handoff Packet: SP-016 / SP-054 / SP-055 / SP-056

## 目的

本パケットは、以下4仕様の現在地と依存関係をまとめ、次の実装担当へ安全に引き継ぐための実務用メモである。

- `SP-016` 拡張テキストボックス機能
- `SP-054` フォント切り替え
- `SP-055` リッチテキスト強化
- `SP-056` 拡張スクロール（キャンバス）

本リポジトリは他Workerの未コミット差分を含む可能性があるため、既存差分を保持しながら最小変更で前進することを前提とする。

---

## 現在の状態

- `SP-054`: 実装差分あり。フォント設定責務の整理と保存処理安全化が主題。
- `SP-055`: 実装差分あり。`RichTextCommandAdapter` と関連ランタイムの導入が進行中。
- `SP-056`: 仕様/設計追加済み。Phase 1 は `single node + pan/zoom + beta flag`。
- `SP-016`: 仕様/設計追加済み。Phase 1 は textarea 先行、WYSIWYG 本格連携は後段。

---

## 依存関係

1. `SP-054` は単独で前進しやすい。
2. `SP-055` は `SP-016` と `SP-056` の共通基盤に近い。
3. `SP-016` は Phase 1 では `SP-055` に深く依存しないが、Phase 2 以降で依存が強くなる。
4. `SP-056` は Phase 1 では `SP-055` の最終形を待たずに進められるが、`fixedTextScale` と WYSIWYG 同期は後段で合流する。

推奨実装順:

1. `SP-054`
2. `SP-055 Phase 1`
3. `SP-016 Phase 1`
4. `SP-056 Phase 1`

並走する場合:

- `SP-055` と `SP-016` は `js/editor-wysiwyg.js` の大改修を同時にやらない
- `SP-055` と `SP-056` は `js/editor.js` / `js/modules/editor/*` の責務境界を守る
- `SP-016` は textarea 側へ寄せる
- `SP-056` は canvas root と viewport 制御へ寄せる

---

## 決定事項

### SP-016

- `sfx` は Phase 1-3 では識別子保存のみ
- 標準プリセットIDは固定、表示ラベルは日本語初期値
- `Ctrl+Alt+1..3` は Phase 1 では採用しない

注意:

- 仕様本文内にショートカット記述が残っていないか、実装前に再確認すること

### SP-055

- `img` は Phase 1-3 のスマートペースト許可対象に含めない
- 見出し操作は H1-H3 に限定
- ペースト初期モードは `smart`
- `settings.editor.richtextEnhanced` を最小ロールバックフラグとして持つ

### SP-056

- 初期リリースは単一ノード
- 回転は Editor 全体回転のみ先行
- タッチ既定操作は `1本指=編集` / `2本指=パン・ズーム`

## 実装境界

### Worker A: SP-054 + SP-055

担当:

- 保存処理安全化
- Typography 責務分離
- RichTextCommandAdapter
- richtext runtime / rollback flag
- WYSIWYG 回帰テスト

避けること:

- Canvas 実装
- Textbox WYSIWYG 本格連携

### Worker B: SP-016

- settings.editor.extendedTextbox
- preset registry
- DSL parser
- textarea での適用導線
- EditorCore 前段連携
- textbox 単体テスト

避けること:

- `js/editor-wysiwyg.js` の大規模改修
- ショートカット実装

### Worker C: SP-056

担当:

- Canvas beta flag
- canvas root / viewport / hud の骨組み
- single node 前提の pan / zoom
- canvas E2E

避けること:

- 複数ノード
- 回転
- fixedTextScale
- WYSIWYG 深い同期
